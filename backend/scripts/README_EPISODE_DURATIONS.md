# Обновление длительностей эпизодов аниме

## Проблема
В разделе "Моя коллекция" отображается "0 часов просмотрено", потому что для расчета часов используются статические значения:
- 24 минуты для сериалов
- 60 минут для фильмов

Это не отражает реальное время просмотра, так как разные аниме имеют разную длительность эпизодов.

## Решение
1. Добавлено поле `episode_duration` в модель `Anime` (в минутах)
2. Обновлена логика расчета часов в `users/views_library.py`
3. Созданы скрипты для получения реальных длительностей из Kodik API

## Скрипты

### 1. Обновление длительностей для существующих аниме
```bash
cd backend
python scripts/update_episode_durations_from_kodik.py [опции]
```

**Опции:**
- `--limit N` - ограничить количество обрабатываемых аниме
- `--batch-size N` - размер батча между паузами (по умолчанию: 10)
- `--shikimori-only` - обрабатывать только аниме с Shikimori ID

**Примеры:**
```bash
# Обновить все аниме (по умолчанию)
python scripts/update_episode_durations_from_kodik.py

# Обновить только 100 аниме
python scripts/update_episode_durations_from_kodik.py --limit 100

# Обновить с батчем по 20 запросов
python scripts/update_episode_durations_from_kodik.py --batch-size 20
```

### 2. Импорт новых аниме с длительностями
Стандартный скрипт импорта `import_from_kodik.py` теперь автоматически определяет и сохраняет длительности эпизодов при импорте новых аниме.

## Как это работает

### Получение данных из Kodik
Kodik API возвращает информацию об эпизодах с параметром `with_episodes_data=True`:
```json
{
  "episodes": [
    {
      "episode": 1,
      "duration": 1425,  // в секундах
      "title": "Episode 1",
      ...
    },
    ...
  ]
}
```

### Алгоритм расчета средней длительности
1. Собираются все длительности эпизодов из Kodik
2. Фильтруются выбросы (менее 30 сек, более 2 часов)
3. Вычисляется медиана для устойчивости к выбросам
4. Конвертируется в минуты
5. Корректируется для типов аниме:
   - Если < 5 минут → 24 мин (стандарт)
   - Если > 180 минут → 60 мин для фильмов, иначе 24 мин

### Использование в расчетах часов
В `users/views_library.py`:
```python
# Старый код
episode_duration = 60 if item.anime.kind == 'movie' else 24

# Новый код
if item.anime.episode_duration:
    episode_duration = item.anime.episode_duration
else:
    episode_duration = 60 if item.anime.kind == 'movie' else 24
```

## Логирование
Скрипт создает лог-файл `episode_durations_update.log` в текущей директории с подробной информацией о процессе.

## Производительность
- Скрипт делает паузы между запросами чтобы не нагружать Kodik API
- По умолчанию: 0.5 сек между запросами, 5 сек после каждых 10 запросов
- Обработка 1000 аниме занимает примерно 15-20 минут

## Мониторинг прогресса
В логах отображается:
```
2024-01-01 12:00:00 - INFO - Обработано 50/500 аниме
2024-01-01 12:05:00 - INFO - Обновлено аниме 123: Наруто - длительность эпизода: None -> 24 мин
2024-01-01 12:05:01 - INFO - Обновлено 5 эпизодов для аниме 123
```

## Что делать после запуска
1. Запустить скрипт обновления длительностей
2. Проверить логи на наличие ошибок
3. Обновить страницу "Моя коллекция" - часы просмотра должны показывать реальные значения

## Примечания
- Для работы нужен действительный `KODIK_API_TOKEN` в настройках
- Скрипт обрабатывает только аниме с `shikimori_id`
- Если данные об эпизодах недоступны, используется стандартная длительность
- Данные кэшируются в базе и не требуют постоянного обновления
